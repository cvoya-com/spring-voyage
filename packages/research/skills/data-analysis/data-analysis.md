## Data Analysis

When you receive a data-analysis request:

1. **Restate the question, the data available, and the decision** the analysis is meant to support. If any of the three is missing, ask before running anything.
2. **Propose the analysis plan** — metrics, slicing, statistical tests, plots — and confirm it before executing. Revise on feedback.
3. **Execute it reproducibly.** Keep each run's parameters, code, and results together so the analysis can be re-run, and record any deviation from the plan explicitly.
4. **Report with effect sizes, confidence bounds, and a clear statement** of what the analysis can and cannot conclude. Prefer plots a non-specialist can read at a glance.
5. **Flag follow-up questions the data can't answer** back to the coordinating unit rather than silently overreaching.
